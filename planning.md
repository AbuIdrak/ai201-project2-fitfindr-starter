### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
looks through the marketplace dataset and returns matching product entries.
**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): ... keywords or semantic descriptors to help find items 
- `size` (str): ... represents the size of items so users can search/filter/sort by size.
- `max_price` (float): ...price datapoint to help users filter/sort by price.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of matching listing dictionaries sorted by their keyword overlap score (highest score first). Each listing dictionary contains the fields: `id`, `title`, `description`, `category`, `style_tags` (list), `size`, `condition`, `price` (float), `colors` (list), `brand`, and `platform`. Items with an overlap score of 0 are completely excluded.
**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->

---Returns an empty list `[]` cleanly without raising an exception. The planning loop detects this empty list, stops execution immediately to prevent cascading errors downstream, and surfaces an error message asking the user to widen their search parameters.

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
returns a number of outfits based on the users requirments and desires. sorted by most matching first onwards. uses an LLM.
**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): ...-A specific listing dictionary representing the thrifted item the user wants to buy.
- `wardrobe` (dict): ...A dictionary representing the user's wardrobe containing an `'items'` key, which maps to a list of individual wardrobe item dictionaries.

**What it returns:**
<!-- Describe the return value -->
A non-empty string containing detailed outfit recommendations, covering visual pairings, layering recommendations, and explicit styling adjustments (like rolling or tucking).

---If `wardrobe['items']` is empty, the tool intercepts this state gracefully instead of raising an exception. It sends a specialized fallback prompt to the LLM requesting general styling ideas, timeless wardrobe staples, and structural aesthetic combinations that naturally complement the `new_item`.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->

---The planning loop acts as a step-by-step conditional state tracker that evaluates the output of each tool before deciding whether to proceed, loop, or terminate.
1. **Step 1:** It executes `search_listings` using parameters parsed from the user's request. It checks the length of the returned list. If the list is empty (`len(results) == 0`), it halts execution immediately, sets a failure flag, prints an error message to the user, and skips all subsequent tool steps.
2. **Step 2:** If listings are found, it saves the top match (`results[0]`) to the session state. It fetches the user's wardrobe dictionary and passes both to `suggest_outfit`. It verifies that a valid string output is generated.
3. **Step 3:** It takes the generated outfit string and passes it into `create_fit_card`. It finishes the loop only when all three steps successfully append their outputs to the session sequence.

## State Management

**How does information from one tool get passed to the next?**
<!-- Describe how your agent stores and accesses state within a session. What data is tracked? How is it passed between tool calls? -->

---The agent maintains an active session dictionary that serves as an in-memory single source of truth across the lifecycle of the user request. The dict is structured as follows:

```python
session_state = {
    "selected_item": None,       # Holds the top dict from search_listings
    "outfit_suggestion": None,   # Holds the str response from suggest_outfit
    "fit_card": None,           # Holds the final str caption from create_fit_card
    "current_step": "INIT"       # Tracks pipeline execution progress
}

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Halts the pipeline loop early. It prints: "We couldn't find any listings matching those filters! Try loosening your size text or expanding your maximum price limit."|
| suggest_outfit | Wardrobe is empty | Detects that the items list is empty, switches to a fallback styling prompt, and instructs the LLM to generate recommendations utilizing generic closet essentials (e.g., plain white tees, classic denim).|
| create_fit_card | Outfit input is missing or incomplete | Checks for missing or whitespace-only strings and safely returns a pre-written template: `"just picked up this vintage {title} on {platform} for ${price} ✨ breakdown coming soon!"*|

## A Complete Interaction (Step by Step)

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
<!-- What does the agent do first? Which tool is called? With what input? -->
The agent processes the initial message and calls search_listings(description="vintage graphic tee", size=None, max_price=30.0). The function scans the dataset, scores items based on keyword matches, sorts them, and returns the list. The loop extracts the best item: {"id": 12, "title": "90s Sonic Youth Band Tee", "price": 25.0, "size": "L", "platform": "Depop", "condition": "Good"} and commits it to session_state["selected_item"].
**Step 2:**
<!-- What happens next? What was returned from step 1? What tool is called now? -->
The planning loop reads session_state["selected_item"]. It calls utils.data_loader.get_example_wardrobe() to load the user's wardrobe dictionary. It then invokes suggest_outfit(new_item=session_state["selected_item"], wardrobe=wardrobe). The LLM reads the wardrobe items alongside the Sonic Youth tee to output a complete outfit suggestion text. This text is saved to session_state["outfit_suggestion"].
**Step 3:**
<!-- Continue until the full interaction is complete -->
The planning loop detects that session_state["outfit_suggestion"] is present and calls create_fit_card(outfit=session_state["outfit_suggestion"], new_item=session_state["selected_item"]). The tool runs the Groq client with a temperature of 1.0 to generate a fresh, authentic social media caption that mentions the tee, the $25 price point, and Depop exactly once. The resulting text is saved to session_state["fit_card"].
